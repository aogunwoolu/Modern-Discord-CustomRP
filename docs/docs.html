Documentation

Complete reference for CustomRP — a modern Discord Rich Presence manager built on Avalonia 11.

Installation

Requirements

  • Windows 10 or later (x64)
  • .NET 10 Desktop Runtime
  • Discord desktop app running on the same machine

Download

Grab the latest CustomRP.Modern-vX.X.X-win-x64.zip from the Releases page, extract the zip, and run CustomRP.Modern.exe. No installer needed.

Tip: Windows may show a SmartScreen warning on first run because the binary is unsigned. Click More info → Run anyway.

First Run

  1. Open the app. The Preset Library tab opens by default.
  2. Search for any app (game, browser, IDE) or switch to the Editor tab to start from scratch.
  3. A Discord Application Client ID is pre-filled. Click Connect — your presence goes live immediately.
  4. To verify, click your avatar in Discord and look for the activity in the Activity Shelf.

Client ID

The Discord Application Client ID field identifies which Discord Application your presence connects through. Leave it blank to use the global default from Settings. Set a unique ID here to show this presence simultaneously with others — Discord displays one activity per Application ID.

Create free applications at discord.com/developers/applications.

Preset Library

The library ships with hundreds of pre-configured known apps. Each entry includes the activity name, large image URL, and category. Click Load to copy the preset into the Editor, then Connect.

You can also save your own presets to the library via Editor → Save to library. User presets live in %APPDATA%\CustomRP.Modern\presets.

Auto-Detect

The Detect button scans all running processes and shows any that match the known-app library. Click Use next to any result to pre-fill the editor.

Auto-Update

Enable Auto-update in the Editor to have the presence update automatically as the tracked process changes.

OptionDescription
Process nameExecutable name to watch, e.g. zen or chrome (without .exe)
StrategyWindow Title — reads the window title bar. Browser URL — reads the address bar via UI Automation (Chromium/Gecko)
Interval (s)How often to poll the process. Default 5 s. Minimum 1 s.
Use faviconAutomatically fetch and display the current site's favicon as the small image

Browser URL Reading

For Chromium-based browsers (Chrome, Edge, Brave, Arc, Thorium) CustomRP reads the address bar via the Windows UI Automation API — no extension needed.

For Gecko-based browsers (Firefox, LibreWolf, Zen Browser) CustomRP tries UI Automation first, then falls back to reading the session recovery file (recovery.jsonlz4) directly from the browser's profile directory. This file is updated roughly every 15 seconds by the browser.

Note: Some Gecko builds with strict process isolation may prevent UI Automation from reading the URL bar. The session-file fallback handles this automatically but has up to a 15 s lag.

Favicon as Small Image

When Use website favicon as small image is checked, CustomRP automatically fetches the current site's icon and uses it as the presence small image.

How it works

  1. CustomRP reads the current URL from the browser.
  2. The favicon is fetched via wsrv.nl — a public image CDN — which converts ICO files to PNG and serves them from a domain Discord's media proxy can reach.
  3. If a Katsau API key is configured in Settings, the Katsau service is used instead to parse the page HTML and find the highest-quality icon available.

The first update cycle after connecting may show no image while the favicon resolves. It appears automatically within one interval (default 5 s).

Get a free Katsau API key at katsau.com for better favicon quality on sites with non-standard icon setups.

Multiple Presences

Discord displays one activity per Discord Application ID. If two presences share the same ID, the newer one replaces the older one in your profile.

To show multiple presences simultaneously (e.g. a game and a browser tab):

  1. Go to discord.com/developers/applications.
  2. Create a new Application for each presence you want to run in parallel. Name them anything — the name is not visible to Discord friends.
  3. Copy the Application ID from the General Information page.
  4. In the CustomRP Editor, paste the ID into the Discord Application Client ID field for each preset.
  5. Connect each one. They will appear as separate entries in Discord's Activity Shelf.
You can create up to 25 free Discord Applications per account. A global Default Client ID can be set in Settings as a fallback for presets that don't specify their own.

Active Presences Panel

The Active Presences tab shows all live connections at a glance: connection state, Discord username, and the last payload sent. From here you can:

  • Reconnect a dropped connection without opening the editor.
  • Stop individual presences independently.
  • See at a glance what Discord is currently displaying for each app.

CustomRP saves the active set to disk on change, so connections are automatically restored the next time the app starts.

Preset Files

Presets are stored as .crpreset files (JSON). Legacy .crp (XML) files from older CustomRP versions can also be imported.

User preset directory

%APPDATA%\CustomRP.Modern\presets\

File format overview

{
  "clientId": "123456789012345678",
  "activityName": "My Game",
  "details": "Level {title}",
  "state": "Online",
  "largeImage": { "key": "https://...", "text": "Playing" },
  "smallImage": { "key": "", "text": "" },
  "buttons": [{ "text": "Watch", "url": "https://..." }],
  "autoUpdate": {
    "enabled": true,
    "strategy": "WindowTitle",
    "processName": "mygame",
    "intervalSeconds": 5
  }
}

Troubleshooting

Presence not showing in Discord

  1. Make sure Discord is running (desktop app, not browser).
  2. Check Settings → Diagnostics in CustomRP for the RPC log path. Open the log to see connection events.
  3. In Discord go to Settings → Activity Privacy and ensure "Display current activity" is enabled.
  4. If another app (e.g. a game with native Rich Presence) is running, it may override yours. Disable its integration in Discord or close it.

Only one presence shows in Discord

Discord stacks multiple activities in the Activity Shelf — click your own avatar to see all of them. The status bar only shows the primary activity. Ensure each presence uses a unique Discord Application ID (see Multiple Presences).

Small image shows a "?" in Discord

  • Discord cannot load the image URL. CustomRP routes favicons through wsrv.nl which Discord can access — if you set a manual image URL, ensure it is a direct HTTPS link to a public image (no redirects required).
  • Portal asset names (e.g. my_icon) only work with images uploaded to your Discord Application in the Developer Portal under Rich Presence → Art Assets.

Browser URL not reading / wrong URL

  • For Chromium browsers, CustomRP uses UI Automation — ensure Discord and CustomRP are running as the same user (not one elevated).
  • For Zen / Firefox, the session file is read from %APPDATA%\zen\Profiles\*\sessionstore-backups\recovery.jsonlz4. This file updates every ~15 s.
  • Check the auto-update.log file in %LOCALAPPDATA%\CustomRP.Modern\ for diagnostic output on every capture cycle.

Legacy CustomRP conflict

Running the old CustomRP (WinForms) at the same time as CustomRP.Modern will conflict on the Discord IPC pipe. Close the legacy version first.

FAQ

Is this safe / will I get banned?

Rich Presence is an officially documented Discord feature. CustomRP uses Discord's public IPC API exactly as described in the documentation. There is no risk of account action.

Does it work without internet?

Yes for everything except the favicon feature (which fetches images from wsrv.nl) and the Katsau API. Core presence functionality is fully local.

Can I use image URLs instead of portal assets?

Yes. Any public HTTPS image URL works in the large/small image fields. Discord proxies it through mp:external automatically.

How do I start CustomRP with Windows?

Add a shortcut to CustomRP.Modern.exe in your Startup folder:

%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup

What data is collected?

None. CustomRP stores all data locally. The only outbound connections are to Discord (IPC, local) and optionally wsrv.nl / Katsau for favicons. See the full Privacy Policy.

Privacy

CustomRP does not collect telemetry, analytics, or usage data of any kind. All configuration is stored locally at:

%APPDATA%\CustomRP.Modern\settings.json
%APPDATA%\CustomRP.Modern\presets\

The following optional outbound connections are made only when you enable the relevant features:

  • wsrv.nl — fetches favicon images when "Use website favicon" is enabled. The current page hostname is included in the request.
  • katsau.com — only if you configure a Katsau API key in Settings.
  • Discord IPC — local named-pipe connection to the Discord desktop app. No network traffic; data never leaves your machine.
